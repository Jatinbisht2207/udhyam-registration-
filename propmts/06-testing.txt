### Goal

Design and implement a complete testing suite for the Udyam Registration Assignment.

The testing suite should validate the existing frontend, backend, and database modules without modifying application logic.

The testing suite should integrate with the existing frontend, backend, and database modules without requiring changes to their implementation.

The tests should ensure the application behaves correctly under valid, invalid, boundary, and failure scenarios.

Do not generate frontend components, backend business logic, database schema, Docker, or deployment configuration.

Only generate testing-related files and configuration.

General Rules

1.

Think like a Senior QA Automation Engineer.

2.

Follow production-grade testing practices.

3.

Generate complete working tests.

4.

Do not modify application logic.

5.

Do not modify frontend or backend business logic.

6.

Do not modify database schema.

7.

Keep tests isolated and repeatable.

8.

Avoid flaky tests.

9.

Use descriptive test names.

10.

Keep fixtures reusable.

11.

Follow Arrange-Act-Assert (AAA) testing pattern.

12.

Clean up test data after execution.

13.

Do not generate Docker or deployment code.

14.

Only generate testing configuration and test files.

15.

Do not invent test cases for features outside the assignment.

16.

Generate tests only for implemented functionality.

17.

Explain why every test suite exists.

18.

After completing the testing module, stop and wait for my confirmation.
